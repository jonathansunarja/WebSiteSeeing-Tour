THE MUSEUM

What is CSS?

A web document is typically built by a triad of components:

  • content
  • presentation,
  • and behaviour.

CSS or Cascading Style Sheets is a style sheet language. It is widely used to describe the styling and presentation of a markup language document. Although it is mostly know to be used with an HTML document, CSS can also be used to set the styling of any XML document. Therefore, with CSS, design and layout can be changed quickly and easily without changing markup in a document. We will discuss more about its usage in the factory.

Brief history of CSS

Style sheets have existed for some time before CSS was first introduced. In the 1980s, there were the early developments of Standard Generalised Markup Language (SGML). However, early style sheet languages like FOSI and DSSSL were not deemed suitable for the web. This was due to the fact that they did not come from different sources in the web.

By the time Hypertext Markup Language (HTML) grew, more stylistic capabilities were introduced to aid designers in having more control over presentation of a website. The major downside to this was the written HTML started to get too complex with more design implementation. Even Robert Cailliau, who was heavily instrumental in the birth of HTML, called for the separation of content and presentation through separate style sheets.


Håkon Wium Lie

Proposal for CSS rose in 1994, around 3 years after HTML was introduced by Tim Berners-Lee. The man behind this was Håkon Wium Lie, who was working with Berners-Lee and Cailliau at CERN around that time. He collaborated with a fellow W3C employee, Bert Bos, to develop CSS. Bos was so influential that he, himself, is widely known as co-author of CSS1 and co-creator of CSS. By 1996, numerous style sheet languages were already proposed and people were drawn into public discussions about uses of style sheet languages for the web. This series of circumstances lead to the World Wide Web Consortium releasing the first W3C CSS Recommendation (CSS1) in December 1996.

Soon after taken into adoption by W3C, CSS was developed together with HTML and the Document Object Model (DOM). Lie went on to integrate CSS into the Arena web browser. Meanwhile, Bos ended up being the chairman of the CSS Working Group years after that. CSS, though, endured turbulent first few years as no web browser could achieve the near-full implementation of the specification. It was only until 2000 that CSS started to kick off.

THE FACTORY

The separation of concerns

As stated at the museum, CSS is responsible for describing the presentation of a document written in a markup language. In full below is how a typical web document is built:

  • Content structure and semantics: HTML
  • Presentation style and layout: CSS
  • Behaviour scripting and interactivity:
    • JavaScript (client-side)
    • PHP, Perl, Java, Ruby, etc. (server-side)

Why is the separation made in the first place? Firstly, this is a good practice to ease developers in maintaining their website, given they can classify easily where an issue relies: content, presentation, or behaviour. Also, this consistency is a huge benefit for a team of developers or when someone has to look into someone else's script. This obviously improves accessibility and control. In theory, separation of content tells us a lot about a page as well. We would be able to differ between a heading and large font, strong emphasis and bold, or a new paragraph and a line break. Generally, the separation of concerns works as how lines of code are broken down into methods that perform different functions. It gives so much ease when elements of a web document are separated into building blocks.

CSS: A simple view

CSS allows visual presentation to be applied to structured mark-up. CSS allows reflowing/positioning of content. This is done through pattern matching syntax, which identifies where to apply style. Value syntax will specify what to apply to a given property. Not only those, external images, animation and transformations can also be applied by using CSS.

Why is styling that important? Simple enough, plain web pages are dull. And when it comes to websites, user experience (meaning aesthetics enhance included) is very crucial. Styling is also a good platform for creative expression. However, overstyling a page would also be a downside. Hence, there needs to be a balance to avoid both monotonous and "noisy" design

External style sheet

External style sheets are stored in separate files. These style sheets will be linked to the markup document in the header section, by referencing to their file directories. Multiple style sheets can be applied to a single page


CSS selectors

There are three basic selector types that define patterns to find in the markup:

Type Definition CSS selector pattern Specificity value
Tag Match all instances of the tag, e.g. every <p>...</p> The tag itself without '<' and '>' 1
Class Match tags containing this class attribute A dot (.) 10
ID Match the unique tag containing this id attribute Hash symbol (#) 100

These selectors can be combined together in forming rules and the final appearance is composite of all appropriate rules. In some cases, there might be conflicting property values, they are solved by these rules:

  1. Source: user-specified styles are more specific
  2. Specificity: relative weighing of selector priority. This is measured by adding up the total specificity value of each rules. The higher the value, the more specific the rule is. For example: a tag inside an ID (total specificity value = 1 + 100 = 101) is more specific than a tag inside a class (total specificity value = 1 + 10 = 11)
  3. Order declared: if specificity value are the same then "last one wins" (means inline styles are always more specific)

CSS units

CSS supports many types of measurement unit:

  • Absolute units calculated independently of other page content and/or browser defaults
    • Useful for precise layout
    • Include pixels (px), points (pt), milimetres (mm)
  • Relative units calculated proportionally against other page content or a browser default
    • e.g. currently available width, default text size, etc.
    • Include percentages (%), Ems (em), Exes (ex)
    • Also special relative units for text (e.g. small, large, x-large)

A good design uses a combination of both.

CSS colours

CSS allows rich control over colour. Any colour can be specified using RGB or Hexadecimal codes. Actually, there are only 17 valid names in CSS2 (e.g. red, magenta, blue, yellow).

Pseudo elements and classes

Pseudo elements refer to selectors for special parts of some elements. For example: p.opening:first-letter allows us to control the first letter in a paragraph with class name opening. Meanwhile, pseudo classes refer to selectors for special status of some elements. For example: a:hover allows us to specify rules on links upon which the cursor hovers.
